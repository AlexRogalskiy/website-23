{{< /blocks/cover >}} {{% blocks/lead color="primary" %}} SuperEdge is an open-source container management system for edge computing. It extends native Kubernetes to the edge in a non-intrusive way. {{% /blocks/lead %}} {{< blocks/section color="gray-dark" >}}

Key Features



{{% blocks/feature icon="fas fa-cloud" title="Cloud Native" %}} Supports all native Kubernetes APIs and resources. {{% /blocks/feature %}} {{% blocks/feature icon="fas fa-cogs" title="Edge Autonomy" %}} L3 edge autonomy for unstable network between cloud and edge hosts. {{% /blocks/feature %}} {{% blocks/feature icon="fas fa-bell" title="Distributed Health Check" url="/docs/reference/edge-health/" %}} Build-in edge-side health monitoring for more accurate health-check to maintain edge autonomy. {{% /blocks/feature %}} {{% blocks/feature icon="fas fa-object-group" title="ServiceGroup" url="/docs/components/servicegroup/" %}} Provides new workload unit to orchestrate container application in a closed-looped network by region. Canary deployment is supported. {{% /blocks/feature %}} {{% blocks/feature icon="fas fa-exchange-alt" title="Network Tunneling" url="/docs/components/tunnel/" %}} Tunnel between kube-apiserver on the cloud and kubelet at the edge. {{% /blocks/feature %}} {{% blocks/feature icon="fas fa-tools" title="Edgeadm Installer" url="/docs/installation/install-via-edgeadm/" %}} Provides edgeadm to bootstrap an edge Kubernetes cluster. {{% /blocks/feature %}} {{< /blocks/section >}} {{< blocks/section >}} {{% blocks/feature icon="fas fa-comments" title="Join community forum!" %}} Visit [SuperEdge Forum](https://groups.google.com/g/superedge) for announcement and technical Discussions. {{% /blocks/feature %}} {{% blocks/feature icon="fab fa-github" title="Contributions are welcome!" %}} Create a [Pull Request](https://github.com/superedge/superedge/pulls) on GitHub to get started.
